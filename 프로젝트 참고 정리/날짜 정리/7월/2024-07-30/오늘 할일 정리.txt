1. 2차 프로젝트를 잠시 뒤로 미루고 1차 프로젝트 정리 작업(진행 중).

1-1. admin 계정 세분화 하고, 각 계정 권한 설정하기(진행 중).
1-2. 예외 처리 할 때 글로벌 예외 처리를 하도록 코드 조정하기(진행 중).
1-3. account 패키지 구조 정리 및 해당 소스 파일 수정(진행 중).
- user, admin 2개로 나누지 않고, user 클래스에서 통합해서 2개를 관리하는 걸로 결정.
- roles 이걸로 user, admin 역할을 나눈다.
1-4. user 엔티티와 관련된 클래스들 정리. admin 추가는 추후에 다루기(진행 중).
- address 클래스 정리
- cart 클래스 정리
- order 클래스 정리
1-5. user 엔티티 클래스에서 userId 필드 username으로 바꾸는 중. 다른 곳도 필드명 처리(완료).
1-6. user와 address 외래키 관계 수정하기(진행 중).
- user를 레퍼지토리에 save를 하고, 해당 user에서 다시 address 값을 가져오고, 그것을 address 레퍼지토리에 save를 한다. 이러면 서로 연결이 된 채로 각각 db에 저장이 될 것으로 예상이 된다.

기타.

orderstatus, gender는 나중에 관련 클래스 레퍼지토리, 서비스 수정할 때 같이 수정.
